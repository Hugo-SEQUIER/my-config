# Content Weekly

Generate content ideas and drafts from recent notes using the ObsidianMemory Content Vault workflow.

## Required paths

- Content root: `/Users/hugo/Documents/ObsidianMemory/Content Vault`
- Workflow: `/Users/hugo/Documents/ObsidianMemory/Content Vault/workflows/generate-content-from-week.md`
- Humanization guide: `/Users/hugo/Documents/ObsidianMemory/Content Vault/guides/seo-humanize.md`
- Content pillars: `/Users/hugo/Documents/ObsidianMemory/Content Vault/guides/content-pillars.md`
- Positioning: `/Users/hugo/Documents/ObsidianMemory/Content Vault/guides/positioning.md`
- Output backlog: `/Users/hugo/Documents/ObsidianMemory/Content Vault/ideas/weekly-ideas.md`

## Workflow

1. Read workflow, content pillars, positioning, and humanization guide.
2. Search recent notes from:
   - Content Vault drafts and idea backlog.
   - Dev Vault project changelogs, kanban, learnings, and decisions.
   - Legacy ObsidianHugo journals/reviews when relevant.
3. Extract 5-10 concrete moments from the week:
   - problem
   - decision
   - result
   - failed attempt
   - lesson
4. Produce:
   - 5-10 post ideas
   - 2 LinkedIn drafts
   - 1 Medium article outline
5. Append a dated section to the output backlog when the user wants persistence.

## Quality bar

- Every idea needs source context or `source needed`.
- Map each idea to a content pillar.
- Prefer real work over generic thought leadership.
- Do not invent what happened during the week.
